# AI Context Pack

This folder contains medium-detail project context intended for AI coding sessions.

## Recommended read order

1. architecture.md
2. workflow-and-safety.md
3. tools-catalog.md
4. testing-and-quality.md
5. backlog-and-roadmap.md

## Why this exists

AI sessions can lose context over long conversations. These files provide stable, repository-local reference for:

- server architecture and boundaries
- safety model and mutation controls
- MCP tool responsibilities and expectations
- test and quality gate behavior
- active backlog and known limitations

## Scope

This context pack is focused on packages/mcp-server.
It intentionally avoids documenting unrelated packages in depth.
